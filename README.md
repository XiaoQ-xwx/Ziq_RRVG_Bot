# 籽青 (Ziq) - Telegram 群组媒体沉淀与分发助理

本工具是一个专为 Telegram 超级群组 (Supergroup) 及其话题 (Topics) 功能打造的媒体收集与分发机器人。基于 Cloudflare Workers + D1 数据库构建，完全 Serverless 化，零服务器成本，为您提供安全、私密且极其顺滑的群组图库管理体验。

## 注意

本项目 99% 都由 AI 编写，使用的模型为：Google Gemini Pro / Claude Opus 4.6 / Codex

## ✨ 核心特性

- **⚡ 极简自动化部署**
  部署后只需在浏览器中访问一次 Worker 域名，系统将全自动完成 D1 数据库建表、字段无感升级以及 Telegram Webhook 的注册绑定，免除一切手动配置烦恼。

- **🏷️ 高度自由的自定义分类**
  不再局限于预设类型！在任意群组话题中发送 `/bind <自定义分类名>`（例如 `/bind 动漫`、`/bind 梗图`），系统会自动将该话题指定为对应图库的数据源。

- **🔒 极度严苛的隐私与群组隔离**
  - **物理隔离**：不同群组的数据及设置表完全独立，互不干扰。
  - **私聊实时查岗**：仅允许群成员在私聊中查看其所在群组的图库。每次抽取前实时核验身份，退群瞬间权限自动剥夺。

- **🏥 智能失效检测与自动愈合**
  针对 Telegram 撤回消息导致的死链问题，独创被动体检机制：抽取到失效图库时将在底层自动剔除并重试；遇到群组解散或机器人被踢出等极端情况，将自动触发"连坐清理"，瞬间清空失效群组的所有关联数据。

- **🎯 智能随机过滤器** *(V5.9 新增)*
  支持按媒体类型（图片/视频/动图）、收录时间（今日/7天/30天/自定义）、视频时长（30秒内/1分钟内/自定义）三维度精准筛选，每用户独立配置。

- **📦 批量操作** *(V5.7 新增)*
  支持批量删除 (`/bd`) 和批量转移 (`/bmv`) 功能，通过转发媒体选择目标，5 分钟超时保护。

- **📜 媒体溯源查询** *(V5.8 新增)*
  回复任意媒体发送 `/list`，即可查看该媒体的所有收录记录与原始消息链接。

- **📱 Telegram Mini App**
  内置 Web App 前端，提供数据看板、收藏夹、历史足迹等可视化 UI。

## 🚀 部署指南

### 1. 准备工作
- 前往 Telegram 联系 [@BotFather](https://t.me/BotFather) 获取 **Bot Token**。
- 将 Bot 邀请至你的目标群组，并**给予管理员权限**。

### 2. Cloudflare 部署
- 创建一个新的 Cloudflare Worker。
- 创建一个 D1 数据库，并在 Worker 的 `Integrations` 中绑定，变量名必须填写为 `D1`。
- 在 Worker 的 `Variables and Secrets` 中添加以下环境变量：
  - `BOT_TOKEN_ENV`: 你的 Telegram Bot Token。
  - `ADMIN_SECRET`: (可选) 自定义密钥，用于外部 Python 脚本导入数据时的安全校验。
- 复制项目内的worker.js的全部代码，在Cloudflare Worker的编辑代码内粘贴并右上角部署。
### 3. 一键初始化
将代码部署至 Worker 后，在浏览器中直接访问你的 Worker 域名（如 `https://your-worker.your-subdomain.workers.dev/`）。页面提示"部署大成功"即代表 Webhook 与数据库已自动就绪。

## 📖 指令列表

### 所有成员可用

| 命令 | 说明 |
|------|------|
| `/start` | 呼出主控制面板（随机抽取、排行榜、收藏夹、历史足迹、设置） |
| `/help` | 查看帮助说明 |
| `/import_json` | 查看历史导入帮助信息 |
| 回复媒体 + `/list` | 查看被回复媒体的所有收录记录与原始消息链接 |

### 管理员专用

| 命令 | 说明 |
|------|------|
| `/bind <分类名>` | 将当前话题绑定为对应分类的采集库 |
| `/bind_output` | 将当前话题设为群内的专属推送展示窗口 |
| 回复媒体 + `/d` | 彻底删除被回复的单条媒体记录 |
| 回复媒体 + `/mv` | 将被回复媒体转移到其他分类（弹出分类选择键盘） |
| `/d <N\|all>` | 批量删除当前分类最近 N 条（无 reply 时触发） |
| `/mv <N\|all> <分类名>` | 批量转移当前分类最近 N 条到指定分类 |
| `/bd` | 进入精确批量删除模式（转发媒体选择，5 分钟超时） |
| `/bd end` | 确认执行批量删除 |
| `/bd cancel` | 取消批量删除会话 |
| `/bmv` | 进入精确批量转移模式（转发媒体选择，5 分钟超时） |
| `/bmv end` | 弹出分类选择键盘，确认执行批量转移 |
| `/bmv cancel` | 取消批量转移会话 |
| 发送 `.json` 文件 + `/import <分类名>` | 内置直导（≤5MB） |

## ⚙️ 群组设置项

通过 `/start` 面板进入设置，每个群组独立配置：

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 展示模式 | B | A=原生转发；B=copyMessage+原始链接 |
| 防重库 | 开启 | 开启后已推送的媒体不再随机抽到 |
| 飞去看看 | 开启 | 抽取成功后是否生成跳转按钮 |
| 重复提示 | 关闭 | 重复收录时是否发消息提示 |
| 成功提示 | 开启 | 抽取成功后是否显示成功提示 |
| 换一个模式 | 替换 | replace=删旧发新；new=保留旧消息 |
| 严格跳过 | 关闭 | false=放回池子；true=强制消耗 |

## 🎯 随机过滤器 (V5.9)

在控制面板中点击"过滤器"按钮，可设置三维度筛选条件：

- **媒体类型**：全部 / 仅图片 / 仅视频 / 仅动图
- **收录时间**：全部 / 今日 / 7天内 / 30天内 / 今年 / 自定义范围
- **视频时长**：全部 / 30秒内 / 1分钟内 / 2分钟内 / 5分钟内 / 自定义上限

每用户独立保存配置，抽取时自动应用。

## 📥 历史数据导入 (均未测试且俺感觉没有xwx)

系统支持两种历史数据迁入方式：

1. **内置直导 (适合 5MB 以内文件)**：
   直接在群内向机器人发送导出的 JSON 文件，并在消息配文中附带 `/import <分类名>` 指令，机器人会自动解析并入库。

2. **外部 Python 脚本 (适合超大库，无限制)**：
   使用项目配套的 `import_history.py` 脚本，在本地进行分批切割推送，绕过 Worker 内存与时长限制（详见脚本内注释）。

## 📊 数据库结构

| 表名 | 说明 |
|------|------|
| `config_topics` | 话题与分类的绑定配置 |
| `media_library` | 媒体记录库（含 duration 列） |
| `user_favorites` | 用户收藏夹 |
| `last_served` | 用户最近抽取记录 |
| `served_history` | 防重库 |
| `chat_settings` | 群组独立设置 |
| `user_history` | 用户浏览足迹（自动保留 50 条） |
| `group_history` | 群组浏览足迹（自动保留 50 条） |
| `user_roster` | 用户花名册 |
| `batch_sessions` | 批量操作会话（5 分钟自动过期） |
| `user_filters` | 用户随机过滤器配置 |
